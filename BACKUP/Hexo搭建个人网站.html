花了好几天，熬了几个夜晚，摸索了好久终于把自己的个人博客网站搭建起来了，顺便来一篇教程，分享给大家，同时也方便以后再次部署。我之前是在Github上搭建了一版，Github+hexo，国内加载图片等等资源奈何实在太卡了，想起自己在阿里云有台服务器，然后直接部署到云上了,下面是我具体的部署的一些步骤。

需要准备的东西

  • 一台个人PC
  • money!(开玩笑的 ^-^)
  • 一台linux服务器，可以买台云服务器，没有又想体验的话就本地跑吧~（或者托管github自己查查怎么弄）

需要的软件环境

  • Git

  • Node.js

  • hexo-cli

  • Nginx

Windows版本

安装hexo

  • 右键开始按钮，打开Windows PowerShell管理员窗口。
  • 执行命令npm install -g hexo进行安装，如下图所示：

  • 安装完成之后，执行命令hexo -v查看是否安装成功。

  • 安装成功之后，右键桌面进入Git终端，选择博客项目需要存放的位置，可以使用mkdir Blog命令来新建一个项目目录。

  • 创建目录成功之后（或者选择你自己的目录），进入对应的Blog目录，执行命令hexo init对项目进行初始化。

  • 在Blog目录下，执行命令hexo g生成demo，执行命令hexo s启动博客项目开启端口监听，启动成功后，可以再浏览器输入localhost:4000进行查看，若有Hello World界面说明成功了。

  • 在Blog目录下，执行命令hexo new "testBlog"生成一个文件叫testBlog.md的Markdown文件，该文件在source/_posts目录下。

  • 打开testBlog.md文件，将以下内容整个复制过去。

  • 执行命令hexo clean && hexo g &&hexo s一次性完成清理、重新生成、启动监听。

  • 在浏览器输入localhost:4000查看文章。

  • 以后需要新增文章，只需要按照这个步骤来进行就可以了，至于文章的编写，具体需去学习如何编写markdown文件。

文章评论系统

目前本站使用的是Valine评论系统，感觉还不错，下面介绍怎么配置使用这个系统。

首先这个Valine评论系统是可以很好的支持Next主题的，它支持Markdown语法，没有后端是基于LeanCloud的所以我们需要注册一个LeanCloud账户，

LeanCloud官网入口地址

  • 设置Next主题，在主题配置文件中找到valine进行配置，将生成的App ID 和 App Key填入对应的字段，如下图。

  • 如果想要管理评论，进入 leancloud 官网，找到 控制台->存储->commet 中进行管理。

  • 如需取消某个 页面/文章 的评论，在 md 文件头部标签增加 comments: false
